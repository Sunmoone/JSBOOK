摘要

进度事件规范定义了一个事件接口，可以用于衡量进展的情况 ；例如 HTTP 实体主体转移。本规范主要是为了使用的其它规格。

1简介

这一节是非规范性的。

此规范定义的事件接口 — — ProgressEvent — — 这可以用于衡量进展情况。其他规格为此目的使用此规范。

在此示例中XMLHttpRequest，结合在本规范中和progress的 HTML 元素中定义的概念是一起使用以显示读取资源的过程。[XHR][HTML] 

<!DOCTYPE html>
<title>Waiting for Magical Unicorns</title>
<progress id="p"></progress>
<script>
  var progressBar = document.getElementById("p"),
      client = new XMLHttpRequest();
  client.open("GET", "magical-unicorns");
  client.onprogress = function(pe) {
    if(pe.lengthComputable) {
      progressBar.max = pe.total;
      progressBar.value = pe.loaded;
    }
  }
  client.onloadend = function(pe) {
    progressBar.value = pe.loaded;
  }
  client.send();
</script>

充分的工作代码当然会加以详细说明和处理更多的方案，例如，网络错误或终止该请求的最终用户。

4接口ProgressEvent

[Constructor(DOMString type, optional ProgressEventInit eventInitDict)]
interface ProgressEvent : Event {
  readonly attribute boolean lengthComputable;
  readonly attribute unsigned long long loaded;
  readonly attribute unsigned long long total;
};

dictionary ProgressEventInit : EventInit {
  boolean lengthComputable;
  unsigned long long loaded;
  unsigned long long total;
};

使用ProgressEvent接口的事件表明一种级数。

lengthComputable属性必须返回它被初始化为的值。当创建事件时该属性必须初始化为 false。

loaded和total属性必须返回他们被初始化为的值。当创建事件属性必须初始化为 0。

4.1射击为 HTTP 使用ProgressEvent接口的事件

对消防命名为e的进度事件意味着对火灾事件名为e使用与事件的ProgressEvent接口，也符合这些条件：

  • 初始化loaded属性到 HTTP实体正文传输的字节数。
  • 如果 HTTP实体正文的长度已知通过Content-Length标头，初始化的lengthComputable属性为真，初始化的total长度属性。

这一定义可以使用的其他规格。客户端代码，例如执行此操作。[XHR]

4.2激发事件的其他上下文中使用的ProgressEvent接口

这是作为练习留给为规范，它引入了这种情况下的编辑器。编辑器是鼓励符合这和其他的规格，利用事件使用ProgressEvent接口的方式来定义它。

4.3建议使用ProgressEvent接口的事件的名称

这一节是非规范性的。

下表概述了与事件使用ProgressEvent接口一起使用的建议的type属性值。规范编辑器还可以自由调整细节到他们的特定情况，虽然大力鼓励，讨论它们的用法与 W3C WebApps 工作组关于public-webapps@w3.org以确保输入从熟悉主题的人。

type属性值说明时间当
loadstart 已开始取得进展。一次。第一次。
progress 在取得的进展。零个或更多。在loadstart后已派出.
error 进程失败。零或一次。最后progress一直派遣之后, 或在loadstart已派遣如果progress并没有派出.
abort 进程被终止。零或一次。
load 级数是成功的。零或一次。
loadend 进展情况已停止。一次。 error之一后abort，或load已派出.

error，abort，并load事件类型是互斥的。

整个 web 平台的error、abort，和load的事件类型，其bubbles和cancelable的属性初始化为 false，所以建议使用ProgressEvent接口的所有事件的一致性这样都做则。

4.4安全注意事项

为跨原产地要求某种形式的选择中，例如跨起源资源共享，已用于事件使用ProgressEvent接口派遣，作为前信息 （如大小） 会显示，否则不能获得。[CORS]
